본 저장소는 [ARC Prize 2025 Kaggle 대회](https://www.kaggle.com/competitions/arc-prize-2025)에 참가하며 연구 및 개발한 솔루션의 진화 과정을 담고 있습니다. ARC(Abstraction and Reasoning Corpus)는 인공지능이 새로운 소량의 예제(few-shot) 환경에서 어떻게 논리적으로 추론하고 문제를 해결하는지를 평가함으로써 일반인공지능(AGI)의 가능성을 측정하는 벤치마크입니다.

## 📁 프로젝트 구조

유지보수와 코드 가독성을 높이기 위해 전체 프로젝트를 다음과 같이 모듈화하여 재구성했습니다:

- `notebooks/` - Kaggle 제출용 스크립트 및 노트북 반복 버전들 (`kaggle_notebook_v1` ~ `v10`)
- `solvers/` - 다양한 전략을 구현한 핵심 문제 해결 로직 (`hybrid_solver.py`, `perfect_solver.py`, `ensemble_solver.py` 등)
- `dsl/` - 프로그램 합성(Program Synthesis)을 위해 자체 정의한 도메인 특화 언어(DSL) 구현체 (`arc_dsl_v1.py` 등)
- `analysis/` - 탐색적 데이터 분석(EDA), 패턴 탐지 및 태스크 심층 분석 스크립트
- `evaluators/` - 검증 데이터셋을 대상으로 솔버의 성능을 평가하는 로컬 스크립트 모음
- `results/` - 로컬 평가 결과 및 JSON 데이터
- `utils/` - 데이터 다운로드, 포맷 검증, Kaggle 제출 등을 돕는 유틸리티 스크립트
- `docs/` - 대회 진행 당시의 일일 로그, 인사이트, 체크리스트 등 문서 아카이브

## 🧠 방법론 및 진화 과정

문제 해결 전략은 버전 1에서 10까지 거치면서 점진적으로 발전했으며, 규칙 기반의 휴리스틱과 프로그래밍 기반 합성(Programmatic Synthesis), 그리고 패턴 추출 방식을 혼합(Hybrid)하는 형태로 진화했습니다.

### 초기 모델 (V1 - V5)
- **직접 매칭(Direct Matching):** 회전, 반전, 전치와 같은 기본적인 기하학적 변환 구현
- **스케일링 & 크롭:** 2x/3x 확대 및 0이 아닌 픽셀 영역만 잘라내는(Crop) 로직 추가
- **색상 연산:** 기본적인 색상 반전, 이동(Shift) 및 모듈러 연산 도입

### 고급 하이브리드 모델 (V8 - V10)
버전 10 (`EnhancedHybridSolver`)은 이 프로젝트의 최종 결과물로, 3단계의 우선순위 파이프라인을 특징으로 합니다:
1. **DSL 프로그램 탐색 (최우선):** 사전에 정의된 기본 연산(기하, 크기 조절, 크롭, 색상 등)을 조합하여, 빠르고 정확하게 정답과 일치하는 프로그램(1~2개 연산 조합)을 탐색합니다.
2. **트랜스폼(Transform) 탐색:** 학습 예제(Training examples)에 대해 검증된 50개 이상의 커스텀 변환 함수를 넓은 범위에서 적용해봅니다.
3. **학습 기반 패턴 추출 (Fallback):** 정확한 프로그램이나 변환을 찾지 못한 경우, 크기 비율, 일관된 색상 맵핑(80% 이상 일치), 그리고 반복되는 패턴 검출(Detect and Extend) 규칙을 추출해 적용합니다.

## 📊 최종 결과

- **Kaggle Public Leaderboard 점수:** `1.67`
  - **순위:** 상위 45% (총 1,456팀 중 약 651등 수준)
  - **1위와의 격차:** 1위 팀(NVARC, 27.64점) 대비 25.97점 차이
- **로컬 평가 결과 (V10 기준):** 정확도(Exact Accuracy)는 `0.0`이었으나, 매우 복잡한 태스크에서 높은 부분 점수(픽셀 일치율 최고 ~98.9%)를 달성하여 접근 방식의 유효성을 부분적으로 확인했습니다.

## 🔍 비판적 분석 및 한계점

방대한 DSL과 하이브리드 탐색 메커니즘을 구축하기 위해 많은 엔지니어링 노력이 들어갔으나, 결과적으로 ARC 벤치마크가 가진 근본적인 난이도를 재확인할 수 있었습니다:

1. **수제(Hand-Crafted) DSL의 한계:** 현재 구현은 기하학 및 색상 연산에 크게 의존합니다. 이는 미리 정의된 변환 규칙에 딱 맞는 문제에는 매우 효과적이지만, 새로운 형태의 추상화, 객체 카운팅, 객체 추적, 혹은 물리 법칙이 적용된 문제에서는 완전히 실패합니다. 기본 연산을 조합하는 탐색 공간은 기하급수적으로 커지므로 Kaggle의 12시간 제한 내에 깊은 수준의 프로그램 합성은 불가능에 가깝습니다.
2. **"Pixel-Perfect"의 혹독함:** `pass@2` 평가 방식은 자비가 없습니다. 98.9%의 픽셀이 일치하더라도 결과는 0점 처리됩니다. 솔버가 대략적인 변환(예: 오브젝트 이동 및 확대)은 올바르게 파악했으나, 미세한 배경 노이즈나 엣지 케이스를 처리하지 못해 실패하는 경우가 많았습니다.
3. **진정한 의미의 '일반화' 부재:** 크기 비율을 학습하거나 1:1 색상 맵핑을 찾는 식의 규칙 기반 패턴 매칭은 진정한 의미의 "추론"이 아닙니다. 진짜 일반화를 달성하기 위해서는 인공지능이 "객체(Object)", "포함 관계(Containment)", "의도(Intent)"를 이해해야 하며, 이는 단순 행렬 조작만으로는 포착하기 어렵습니다.

## 🚀 향후 발전 방향 (Next Steps)

최상위권(점수 ~27%) 도달 및 Grand Prize 기준(85%)을 달성하기 위해서는 엄격한 프로그래밍 기반 탐색 방식에서 뉴로-심볼릭(Neuro-symbolic) 또는 파운데이션 모델 방식으로의 전환이 필수적입니다:
- **Test-Time Training (TTT) / 미세조정:** 테스트 예제 자체를 타겟으로 가벼운 Vision Transformer (ViT)나 1D 시퀀스 모델을 훈련하는 방법.
- **LLM 기반 프로그램 합성:** 컴퓨팅 자원이 허락한다면, Open-weights 기반의 언어 모델을 활용해 Python 코드를 동적으로 생성하고 테스트하는 방법.
- **객체 중심 표현(Object-Centric Representations):** 원본 Numpy 그리드 행렬을 직접 조작하는 방식에서 벗어나, 입력 공간을 그래프(Graph)나 객체 지향적인 형태로 추상화한 뒤 변환하는 방식 도입.
